# Міграція www-vorote-ch з Next.js + TinaCMS на Astro

**Goal:** Перенести професійний блог та портфоліо vorote.ch з Next.js/TinaCMS на Astro, позбувшись зовнішньої залежності від TinaCMS.

**Architecture:** Astro (static-first, islands) + Content Collections (MDX) + Tailwind CSS + React islands для інтерактивних компонентів (scheduler, security-audit). Деплой на Cloudflare Workers через офіційний Astro adapter.

**Tech Stack:** Astro 5, @astrojs/cloudflare, @astrojs/mdx, @astrojs/react, Tailwind CSS 4, React 18 (islands only)

---

## Поточний стан проєкту

### Контент
- **12 блог-постів** (MDX + 1 MD) у `content/posts/2025/` та `content/posts/2026/`
- **2 сторінки** (home, about) у `content/pages/` — використовують block-based схему TinaCMS
- **5 milestones** (MD) у `content/milestones/`
- **1 author** (MD) у `content/authors/`
- **14 тегів** (MDX) у `content/tags/`
- **Global config** (JSON) — header nav, footer social links
- **~6MB медіа** у `public/uploads/`

### Сторінки та маршрути
| Маршрут | Призначення |
|---------|-------------|
| `/` | Home — hero, callout, recent posts, latest posts list |
| `/about` | About — block-based MDX сторінка |
| `/posts` | Blog list — всі пости |
| `/posts/[...urlSegments]` | Індивідуальний пост |
| `/tools` | Tools hub |
| `/tools/scheduler` | On-call scheduler (React interactive) |
| `/tools/security-audit` | Security audit tool (React interactive) |
| `/[...urlSegments]` | Catch-all для MDX сторінок |

### TinaCMS залежності (що прибираємо)
- `tina/` — config, collections, generated client, schemas
- `tinacms` пакет
- `@tinacms/cli`
- GraphQL backend на port 4001
- Tina Cloud залежність (client ID, token)

### Компоненти (48 файлів .tsx/.ts)
- **Layout:** header, footer, layout-context, section
- **Blocks:** hero, callout, features, stats, content, CTA, testimonial, video, mermaid, recent-posts, latest-posts-list, journey, table-of-contents, featured-link
- **MDX components:** mdx-components, raw-renderer
- **UI:** button, card, avatar, code-block, hero-video-dialog, etc.
- **Magic UI:** grid-pattern, meteors, script-copy-btn
- **Motion primitives:** animated-group, infinite-slider, progressive-blur, text-effect
- **Tools (React interactive):** scheduler (7 файлів), security-audit
- **Other:** feedback-form, error-boundary, theme-toggle, icon

### API Routes
- `app/api/feedback/route.ts` — відправка фідбеку через Resend
- `app/api/security-audit/route.ts` — security audit API

**Objective:** Замінити TinaCMS schema на Astro content collections з typed frontmatter.

**Files:**

**Step 1:** Створити `src/content/config.ts`
```typescript
import { defineCollection, z } from 'astro:content';

const posts = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
    heroImg: z.string().optional(),
    excerpt: z.string().optional(),
    author: z.string().default('Dmytro'),
    date: z.coerce.date(),
    tags: z.array(z.string()).optional(),
  }),
});

const pages = defineCollection({
  type: 'content',
  schema: z.object({
    title: z.string(),
  }),
});

const milestones = defineCollection({
  type: 'data',
  schema: z.object({
    title: z.string(),
    year: z.string(),
    description: z.string(),
    position: z.number(),
  }),
});

const authors = defineCollection({
  type: 'data',
  schema: z.object({
    name: z.string(),
    avatar: z.string().optional(),
  }),
});

const tags = defineCollection({
  type: 'content',
  schema: z.object({
    name: z.string(),
  }),
});

export const collections = { posts, pages, milestones, authors, tags };
```

import { getCollection } from 'astro:content';

export async function getStaticPaths() {
